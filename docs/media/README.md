# Mídia da vitrine

Esta pasta reserva os arquivos visuais usados no README principal do Omini. As imagens e GIFs devem mostrar o produto em uso, com dados de desenvolvimento e sem informações sensíveis.

## Arquivos esperados

| Arquivo | Tipo sugerido | Conteúdo |
| --- | --- | --- |
| `01-login.png` | Imagem | Tela de login com logo e mascotes do Omini. |
| `02-dashboard-alertas.gif` | GIF | Busca no painel, paginação e alertas de estoque baixo ou sem estoque. |
| `03-cadastro-produto.gif` | GIF | Cadastro de produto com tipo, fornecedor, validade, localização e controle pela PF. |
| `04-retirada-produto.gif` | GIF | Seleção de produto, validação de quantidade, motivo e confirmação de retirada. |
| `05-relatorios.png` | Imagem | Tela de relatórios com tabela, gráfico ou cards de relatório. |
| `06-fornecedores-usuarios.png` | Imagem | Gestão de fornecedores e usuários. |

## Roteiro de captura

1. Subir banco, backend e frontend seguindo o README raiz.
2. Abrir `http://localhost:5173`.
3. Usar dados de desenvolvimento ou registros fictícios.
4. Capturar telas em largura desktop, preferencialmente 1440x900.
5. Cortar apenas barras do navegador ou informações irrelevantes.
6. Salvar os arquivos com os nomes exatos listados acima.
7. Remover os comentários da seção "Demonstração visual" do README raiz para exibir os assets.

## Boas práticas

- Priorize fluxos curtos, com até 10 segundos por GIF.
- Evite texto pequeno demais; a imagem deve continuar legível no GitHub.
- Não inclua credenciais, emails reais, tokens, dados pessoais ou dados institucionais sensíveis.
- Prefira PNG para telas estáticas e GIF apenas para interações que adicionem valor.
